# Web SSH 终端指南

[English](./en/webssh.md)

R-Bot 客户端内置了完整的 Web SSH 终端，通过浏览器即可连接和管理远程服务器，无需安装任何客户端软件。

---

## 访问方式

启动客户端后，访问：

```
https://你的IP:9527
```

> 默认使用 HTTPS + TLSv1.3，首次访问时浏览器会提示证书不受信任，可手动信任或配置 ACME 自动证书。

---

## 登录认证

支持两种登录方式：

### 密码登录

使用 `client_config` 中配置的 `username` 和 `password` 直接登录。

### Telegram 验证码登录

1. 点击「发送验证码」
2. 机器人会通过 Telegram 发送 8 位数字验证码
3. 在 60 秒内输入验证码完成登录

> 防护机制：10 分钟内连续请求超过 3 次将触发图片验证码（算术题），防止暴力攻击。

## SSH 连接

### 新建连接

1. 点击侧边栏「+」或「新建连接」
2. 填写连接信息：
   - **主机** — IP 地址或域名
   - **端口** — 默认 22
   - **用户名** — SSH 登录用户
   - **认证方式** — 密码或私钥
3. 点击「连接」

### 认证方式

| 方式 | 说明 |
|------|------|
| **密码** | 输入 SSH 密码 |
| **私钥** | 粘贴私钥内容或选择已保存的密钥，支持 passphrase |

### 主机指纹验证

首次连接会显示 SHA256 主机指纹，确认后保存。后续连接如果指纹变更（可能是服务器重装或安全风险），会发出警告。

### 多标签页

支持同时打开多个 SSH 连接，每个连接独立一个标签页，可自由切换。

## SFTP 文件管理

连接 SSH 后，点击工具栏的「SFTP」按钮打开文件管理面板。

### 功能列表

| 操作 | 说明 |
|------|------|
| **浏览目录** | 点击文件夹进入，面包屑导航支持快速跳转 |
| **上传文件** | 支持分片上传，显示进度 |
| **下载文件** | 流式传输，512KB 分片 + 滑动窗口确认 |
| **创建目录** | 在当前路径下创建新文件夹 |

## 端口转发

在终端连接状态下，可配置 SSH 端口转发。

### 本地转发（Local Forward）

将远程服务映射到本地端口，例如访问远程数据库：

| 参数 | 示例 |
|------|------|
| 绑定地址 | `127.0.0.1` |
| 绑定端口 | `3306` |
| 目标地址 | `127.0.0.1` |
| 目标端口 | `3306` |

### 远程转发（Remote Forward）

将本地服务暴露到远程服务器端口。

## 批量命令

向多个已连接的会话同时发送相同命令，适用于批量运维场景。

1. 点击工具栏「批量命令」
2. 选择目标会话（可多选）
3. 输入命令并发送

## 会话管理

### 保存会话

连接成功后可将当前配置保存为会话，下次直接点击即可连接。保存的信息包括：

- 主机、端口、用户名
- 认证方式和凭据（加密存储）
- SFTP 默认路径

### SSH 密钥管理

集中管理所有 SSH 私钥：

- 添加/删除密钥
- 连接时自动匹配已保存的密钥
- 密钥使用 AES 加密存储

## SSL 证书配置

### 内置自签证书

客户端默认使用自签 PKCS12 证书，开箱即用。

### ACME 自动证书（Let's Encrypt）

支持自动签发和续期 Let's Encrypt 证书，需要：

1. 一个指向服务器的域名
2. 80 端口可访问（用于 HTTP-01 验证）

在 Web 界面的「设置」中配置：

| 参数 | 说明 |
|------|------|
| **域名/IP** | 证书绑定的域名 |
| **邮箱** | Let's Encrypt 通知邮箱 |
| **验证端口** | HTTP-01 验证端口，默认 80 |

配置后自动签发证书，到期前 2 天自动续期。
